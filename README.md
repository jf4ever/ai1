# Реализация MVP-движка сценариев

Сделал рабочую реализацию ядра, которое можно встроить в Android-приложение с overlay:

- один активный сценарий одновременно;
- поиск первого этапа у всех сценариев, когда активного нет;
- этапы `TemplateTap` (поиск + «человеческий» тап) и `Scroll`;
- тайм-аут каждого этапа с остановкой сценария;
- шум: случайная задержка и смещение точки клика.

## Структура

- `overlay_engine/engine.py` — доменные модели + state machine движка.
- `tests/test_engine.py` — тесты арбитража, завершения и тайм-аута.

## Запуск тестов

```bash
pytest -q
```
